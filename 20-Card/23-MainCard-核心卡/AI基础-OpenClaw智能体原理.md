# AI基础-OpenClaw智能体原理

## 八大核心模块

| 模块 | 职责 | 说明 |
|------|------|------|
| **Gateway** | 网关 | 所有指令的第一道关卡，负责身份验证、安全隔离、指令路由 |
| **Channel** | 通道 | 用户与 AI 助理通信的万能传话通道，支持网页/微信/飞书等 |
| **Agent** | 智能体 | 决策中枢，负责理解意图（观察→思考→规划→执行→反思） |
| **Skills** | 技能 | 万能工具箱，真正执行任务（发邮件、写代码、操作浏览器等） |
| **Memory** | 记忆 | 双层记忆：短期（对话历史）+ 长期（Markdown/向量库语义检索） |
| **Node** | 节点 | 部署在各设备的客户端代理，实现跨设备远程操作 |
| **Heartbeat** | 心跳 | 后台守护进程，主动巡逻、监控、提醒（被动→主动） |
| **Cron** | 定时 | 定时执行周期性任务（日报、周报、月提醒等） |

## 核心关系

```
Agent 负责决定做什么，Skills 负责搞定怎么做
```

**双层记忆**：短期记住对话上下文，长期存储 Markdown 文件和向量库，支持快速语义检索。

## 完整工作流程

1. 用户发送指令
2. Gateway 验证身份并安全路由
3. Agent 进行思考、规划和拆解
4. Skills 真正动手执行
5. Node 跨设备远程操作
6. Memory 默默记录用户习惯
7. Heartbeat 在后台持续巡逻
8. Cron 定时执行周期性任务

八大模块协同配合，最终实现真正的自动化闭环。

## 新一代智能体特征

| 特征 | 说明 |
|------|------|
| 会思考 | 具有规划和决策能力 |
| 有记忆 | 能够学习和记住用户习惯 |
| 懂执行 | 拥有丰富的技能工具箱 |
| 会提醒 | 主动监控和通知 |
| 跨设备 | 实现多设备联动 |
| 永远在线 | 24 小时持续服务 |
| 越用越懂 | 不断学习和优化 |

**核心理念**：把智能体当做自己的数字分身，让它帮你完成工作自动化。
