# 每日开场 Skill

帮助用户在一天开始时确定"今日重点"。

**语气**：简洁直接，不做仪式感包装。

---

## Obsidian CLI

本 skill 通过 Obsidian CLI 读写日志和扫描任务，减少手动路径拼接。

**调用方式**：通过 Bash 执行 `/Applications/Obsidian.app/Contents/MacOS/obsidian <command>`

**常用命令**：

| 命令                                  | 用途               |
| ------------------------------------- | ------------------ |
| `daily:read`                          | 读取今日日志内容   |
| `daily:append content="文本"`         | 在今日日志末尾追加 |
| `read file="文件名"`                  | 读取指定文件       |
| `create name="名称" template="模板"`  | 从模板创建文件     |
| `search query="关键词" limit=N`       | 全库搜索           |
| `files folder="路径"`                 | 列出文件夹内文件   |
| `recents`                             | 最近打开的文件     |
| `append file="文件名" content="文本"` | 在指定文件末尾追加 |

**规则**：CLI 调用失败时回退到文件系统直接操作（Read/Write/Edit），不中断流程。

---

## 核心规则

1. **轻量**：整个流程 2-3 分钟内完成，不做过多展开
2. **只问一个核心问题**：今天最重要的一件事是什么？
3. **不替代日志写作**：这个 skill 只处理"今日重点"字段，日志正文由用户口述完成
4. **不替代周回顾**：不做目标对齐检查，那是 日常-周回顾 的工作

---

| 关键词 | 模式 |
|--------|------|
| "开始今天"、"今天开始" | 开场模式 |

## 日期和文件定位

- 日志路径：`30-Time/34-Daily-日志/YYYY-MM-DD（周几）.md`
- 周记路径：`30-Time/33-Weekly-每周/YYYY-Wxx.md`
- 日志模板：`40-storage/42-template-模板/time-日志.md`
- 星期映射：周一、周二、周三、周四、周五、周六、周日

## 开场模式

### Step 1: 定位今日日志

1. 计算今天的日期和星期
2. 尝试通过 CLI 读取今日日志：`obsidian daily:read`
3. 如果成功，说明日志已存在，继续下一步
4. 如果失败或返回空，在 `30-Time/34-Daily-日志/` 中 glob 匹配 `YYYY-MM-DD*`
5. 如果仍不存在，从模板创建：`obsidian create name="YYYY-MM-DD（周几）" path="30-Time/34-Daily-日志" template="time-日志"`（如 CLI 创建失败，回退到手动读取模板并 Write 创建）

### Step 2: 加载上下文（并行读取）

以下 CLI 调用和文件读取并行执行：

1. **今日日志**：`obsidian daily:read`（已在 Step 1 获取，直接复用）
2. **昨日日志**：通过 `obsidian search query="今日重点" limit=3 path="30-Time/34-Daily-日志"` 找到最近几天的日志，读取最近一篇（或 glob 回退）
3. **当前周记**：`obsidian read file="YYYY-Wxx"`（如有）
4. **AI 记忆文件**（`.claude/memory/[昨日日期].md`，如存在）— AI 上次会话备忘
5. **Todoist 今日/超时任务**（与上述并行）— 通过 Bash 调用 Todoist API 获取
6. **最近活跃文件**：`obsidian recents` — 获取最近打开的文件，辅助判断用户近期关注焦点

### Step 3: 展示简要上下文

简短展示：

- 昨天的"今日重点"是什么（如果有），完成了吗
- 当前周记中有没有本周计划或"下周最重要的一件事"

如果昨天没有日志或没有"今日重点"字段，跳过，不做追问。

### Step 3.5（可选）: 快速待办扫描

在展示上下文之后，快速扫描活跃任务中可以顺手完成的小事：

1. 通过 CLI 列出活跃任务：`obsidian files folder="10-Action/12-Active-活跃跟进"`
2. 读取每个任务文件的前 20 行（标题 + 开头描述）
3. 筛选出预计 15 分钟内可完成的小任务（根据描述判断）
4. 如果有，在上下文之后展示（不超过 3 条）：

```
碎片时间可做：
- T日期-描述
- T日期-描述
```

**规则**：

- 没有快速待办就跳过，不说"没有找到"
- 不影响主流程速度，扫描和 Step 2 并行执行
- 这只是参考信息，不是今日重点的候选项

### Step 4: 确定今日重点

用 AskUserQuestion 问用户：

**问题**："今天最重要的一件事是什么？"

选项基于上下文生成（2-3 个建议 + 自由输入）：
- 如果周记有本周计划，从中提取建议
- 如果昨天有未完成的重点，作为一个选项
- 始终保留自由输入选项

### Step 5: 写入日志

将用户的回答写入今日日志的"今日重点"字段。

**写入方式**（按优先级）：

1. **优先用 Edit 工具**：精确替换"今日重点："后面的内容（最可靠）
2. **CLI 辅助**：如果日志是新创建的空模板，可用 `obsidian daily:append content="今日重点：用户的回答"` 追加

```markdown
今日重点：用户的回答
```

输出确认：

```
已写入今日重点：XXX
日志文件：[[YYYY-MM-DD（周几）]]
```

---

## 注意事项

1. **不要过度展开**：这个 skill 的价值在于快速，不在于深度。深度留给周回顾
2. **今日重点只写一件事**：不是待办清单，是"如果今天只能做一件事，是哪件"
3. **文件创建遵循命名规范**：`YYYY-MM-DD（周几）.md`，星期用中文全称

---

## 与其他 Skill 联动

- **日常-每日收场**：开场确定重点，收场回顾重点并做提取。`开始今天` → 工作 → `收尾今天`
- **日常-周回顾**：周回顾时，每天的"今日重点"是最好的周摘要来源
- **日常-日志想法提取**：日志内容后续可被提取为独立想法
- **记忆系统**：开场的 Step 2 读取昨日 AI 记忆文件补充上下文
