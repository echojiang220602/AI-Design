# 日常-md 重命名 Skill

MD 文件批量重命名的标准操作流程。**重命名 + 同步更新引用** 是一体两面，缺一不可。

---

## 核心规则

### 命名格式

```
<前缀>-<4位编号>-<文件名>.md
```

| 部分 | 含义 | 规则 |
|------|------|------|
| **前缀** | 文件所在**最深文件夹**的数字前缀 | 与文件夹编号一致（如 `1404-Skill广场/` → `1404`） |
| **4 位编号** | 顺序号 | 固定 4 位（0001-9999），按指定规则编号 |
| **文件名** | 去掉原前缀的有意义名称 | 单词间用 `-` 分隔（kebab-case），中文保留 |

### 强制联动：重命名 = 更新引用

**重命名一个文件，必须同步更新所有引用该文件的笔记**。否则 wikilink 失效、双链断裂。

## 标准操作流程（SOP）

### Step 0: 确认排序规则（如果用户未指定）

用 AskUserQuestion 询问排序规则：

1. **按创建时间**（最常用，旧→新 或 新→旧）
2. **按当前文件名**（ASCII 排序）
3. **按内容主题**（需要读文件判定）
4. **按字母/中文拼音**（可预测）

---

### Step 1: 制定重命名映射表

输出明确的映射清单（让用户可审核）：

```markdown
| 新名 | 原名 | 排序依据 |
|------|------|---------|
| 1404-0001-Skill清单.md | 1401-01-SkillHub清单.md | Jun 4 创建 |
| ... | ... | ... |
```

---

### Step 2: 重命名前先 grep 全 vault 引用

**必须先 grep 后再 mv**，避免断链窗口期。

```bash
grep -rn "<旧名>" --include="*.md" /Users/echo/sjld/AI产品设计/
```

引用检测的 5 种 pattern：

| 引用形式 | 写法示例 | grep pattern |
|---------|---------|-------------|
| **wikilink 无显示文本** | `[[旧文件名]]` | `\[\[<旧名>\]\]` |
| **wikilink 有显示文本** | `[[旧文件名\|显示文本]]` | `\[\[<旧名>\|` |
| **纯文本引用** | `旧文件名.md` | `<旧名>\.md` |
| **代码块引用** | `` `旧文件名.md` `` | `` `<旧名>\.md` `` |
| **YAML frontmatter** | `relate: [旧文件名.md]` | YAML 字段值 |

**输出引用位置清单**：

```markdown
| 文件 | 行号 | 类型 | 当前引用 |
|------|------|------|---------|
| index.md | 16 | wikilink | [[260622-...]] |
| 2026-06-17.md | 21 | 纯文本 | `skill-...V1.2.md` |
| 1404-0015-...md | 11 | YAML | relate: 1405-...md |
```

---

### Step 3: 执行 mv 重命名

```bash
cd "<目标文件夹>" && \
mv "旧文件名" "新文件名" && \
mv "旧文件名2" "新文件名2" && ...
```

或者用链式 `&&` 一次性执行。

---

### Step 4: 批量更新所有引用

**对每个引用文件**，按引用类型分别处理：

#### wikilink 更新
- `[[旧名]]` → `[[新名]]`
- `[[旧名\|显示文本]]` → `[[新名\|显示文本]]`（**保留显示文本**）

#### 纯文本更新
- `旧名.md` → `新名.md`

#### YAML 字段更新
- `relate: 旧名.md` → `relate: 新名.md`

**推荐方式**：用 Edit 工具按文件逐处修改（可读、可审、可回退）。

**批量方式**（慎用，需精确 pattern）：
```bash
sed -i '' "s|\[\[<旧名>\]\]|[[<新名>]]|g; s|<旧名>\.md|<新名>\.md|g" <文件路径>
```

---

### Step 5: 验证旧名零残留

```bash
grep -rn "<旧名>" --include="*.md" /Users/echo/sjld/AI产品设计/
```

**期望结果**：无输出（grep 退出码 1）。

如果还有残留，逐个修复后再次验证。

---

### Step 6: 记录经验（可选但推荐）

按 README.md 约定：
- 重要经验写入 `.claude/MEMORY.md`
- 文件夹操作经验写入该文件夹 `README.md` 的 `## Claude 经验积累`
- 临时记录写入 `.claude/memory/YYYY-MM-DD.md`

## 命名细节规则

### 文件名风格

| 类型 | 规则 | 示例 |
|------|------|------|
| 英文单词 | kebab-case（连字符分隔） | `Skill-hub`、`AI-model` |
| 中文 | 保留中文原样 | `技能商城`、`Skill清单` |
| 混合 | 中文为主，英文单词 kebab-case | `AI数字人中台Skill清单` |
| 版本号 | 用 `-` 连接 | `V1.2` → `-V1.2`（保留版本号作为后缀） |
| 括号 | 用 `-` 替代空格 | `技能商城(五) V1.2` → `技能商城(五)-V1.2` |

### 不参与重命名的文件

| 文件 | 原因 |
|------|------|
| `index.md` | 目录索引，固定名 |
| `README.md` | 约定文件，固定名 |
| frontmatter 等结构文件 | 结构性约束 |

---

## 易踩的坑

| 坑 | 后果 | 防范 |
|----|------|------|
| **先 mv 再 grep** | 断链窗口期，其他笔记点不进去 | **必须先 grep 后再 mv** |
| **只改 wikilink，漏纯文本** | 列表项里 `旧名.md` 未更新 | 5 种 pattern 全部 grep |
| **YAML 字段遗漏** | `relate`/`相关`/`index` 未更新 | 检查 frontmatter |
| **跨文件夹引用未发现** | 引用在完全无关的笔记里 | **全 vault grep**，不限文件夹 |
| **显示文本丢失** | `[[旧名\|显示文本]]` 改成 `[[新名]]` 丢了显示文本 | 用精确 pattern `\[\[<旧名>\|` |
| **特殊字符未转义** | 括号、空格在 sed 中报错 | 用 Edit 工具逐处修改 |
| **误改无关文件** | 短文件名匹配到了其他笔记 | grep 后**人工复核**每个匹配 |

## 适用场景

- 文件夹内文件命名混乱，需要规范化

- 命名规则升级（如从 `1401-xx-` 改为 `1404-xxxx-`）
