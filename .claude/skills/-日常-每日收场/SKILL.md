# 每日收场 Skill

一天结束时的 5 分钟结构化收尾。不是写长报告，而是快速提取、连接、归档。

**语气**：简洁直接，重在发现联系而非总结。

---

## Obsidian CLI

本 skill 大量使用 Obsidian CLI 获取 vault 关系数据。

**调用方式**：通过 Bash 执行 `/Applications/Obsidian.app/Contents/MacOS/obsidian <command>`

**常用命令**：

| 命令                                  | 用途               |
| ------------------------------------- | ------------------ |
| `daily:read`                          | 读取今日日志内容   |
| `daily:append content="文本"`         | 在今日日志末尾追加 |
| `tags counts sort=count`              | 按频率列出标签     |
| `search query="关键词" limit=N`       | 全库搜索           |
| `backlinks file="文件名"`             | 文件的反向链接     |
| `links file="文件名"`                 | 文件的外向链接     |
| `orphans`                             | 无入链的孤立笔记   |
| `recents`                             | 最近打开的文件     |
| `files folder="路径"`                 | 列出文件夹内文件   |

**规则**：CLI 调用失败时回退到文件系统直接操作，不中断流程。

## 触发条件

| 关键词                         | 说明     |
| ------------------------------ | -------- |
| "收尾今天"、"今天结束了"       | 收场模式 |
| "close day"                    | 同上     |

---

## 核心规则

1. **5 分钟仪式**：快速提取和归档，不写长报告
2. **重发现联系，不重总结**：今天的笔记是发现模式和联系的入口
3. **不替代日志想法提取**：本 skill 做轻量提取和建议，深度提取留给 `日常-日志想法提取`
4. **所有修改需确认**：建议归档、添加链接等操作，先展示再执行

---

## 执行流程

### Step 1: 读取今日日志

1. 通过 CLI 读取今日日志：`obsidian daily:read`
2. 如果 CLI 失败，回退到 glob 匹配 `30-Time/34-Daily-日志/YYYY-MM-DD*` 后 Read
3. 从内容中提取"今日重点"字段

如果今日日志不存在，告诉用户没有找到今日日志，询问是否要直接做 Quick Wrap（跳到 Step 6）。

### Step 1.5: Vault 连接发现（并行执行）

以下 CLI 调用与 Step 1 的日志解析并行：

1. **活跃主题**：`obsidian tags counts sort=count` — 识别高频标签/主题
2. **今日日志的链接**：
   - `obsidian links file="YYYY-MM-DD（周几）"` — 今日外向链接
   - `obsidian backlinks file="YYYY-MM-DD（周几）"` — 今日反向链接
3. **主题搜索**：对日志中提到的 2-3 个关键主题执行 `obsidian search query="主题" limit=5` — 找到相关笔记
4. **最近活跃**：`obsidian recents` — 今天实际在看的文件
5. **孤立笔记**：`obsidian orphans` — 查看是否有与今日主题相关的孤立笔记

**联系洞察**（基于搜索结果生成）：

- 发现重复出现的主题："[主题] 在最近两周出现了 N 次"
- 发现联系："今天的 [话题] 和 [[某笔记]] 有关联"
- 发现矛盾："之前你写过 [X]，今天写了 [Y]，值得对照看看？"

---

### Step 2: 今日重点回顾

用 AskUserQuestion 问用户：

**问题**：展示今日重点内容，然后问"这个重点做到了吗？"

选项：

- 做到了
- 部分做到
- 没做到
- 今天做了其他更重要的事

如果没有"今日重点"字段，跳过此步，直接进入 Step 3。

---

### Step 3: 提取与分类

从今日日志内容中提取四类信息：

#### 行动项

- 对别人的承诺
- 需要跟进的事
- 截止日期

#### 想法与洞察

- 工作模式观察
- 内容/项目创意
- 认知转变

#### 人与承诺

- 需要回复的人
- 需要安排的会议
- 需要发送的消息

#### 待解决问题

- 需要调查的事
- 待做的决定
- 未解决的不确定性

**展示格式**：

```
## 今日提取

行动项：
- XXX

想法与洞察：
- XXX

人与承诺：
- （如果没有则省略此类）

待解决问题：
- （如果没有则省略此类）
```

如果日志内容简短，某些类别可能为空，直接省略。

---

### Step 4: 归档建议

对 Step 3 提取的每个条目，建议归档位置：

| 条目 | 类型 | 建议位置 | 操作 |
| ---- | ---- | -------- | ---- |
| XXX  | 行动 | `10-Action/12-Active-活跃跟进/` | 创建任务文件 |
| YYY  | 想法 | `+/` 收件箱 | 创建笔记 |
| ZZZ  | 洞察 | 追加到 `[[已有笔记]]` | 在末尾追加 |

**反向链接建议**：

通过 CLI 检查日志中提到的人名、项目、概念是否有对应笔记：

```bash
obsidian search query="人名/项目名" limit=3
```

如果有对应笔记但日志中没有链接，建议添加：
"建议在日志中添加链接：[[人名]]、[[项目名]]"

**规则**：

- 只展示建议，不自动执行
- 用户确认后才执行归档操作
- 如果提取内容少（≤2 条），可以跳过表格，直接用文字说明

---

### Step 5: 写入收尾记录

将回顾结果写入今日日志和 AI 记忆文件。

#### 5a: 追加到日志

通过 CLI 追加：`obsidian daily:append content="收尾内容"`

如果 CLI 失败，回退到 Edit 工具。

```markdown
---

收尾：今日重点「XXX」— 用户的回答
```

不标记 `已回顾: true`，这个留给 日常-日志想法提取。

输出确认：

```
收尾已写入：[[YYYY-MM-DD（周几）]]
```

#### 5b: 写入 AI 记忆文件

将今日摘要写入 `.claude/memory/YYYY-MM-DD.md`（如已有内容则追加，无则新建）：

```markdown
## 今日摘要

- **今日重点**: 用户的今日重点
- **完成情况**: 做到了 / 部分做到 / 没做到 / 做了其他更重要的事
- **关键提取**: （从 Step 3 提取的 1-3 个最重要条目）
- **明日承接**: （从 Step 6 的结果）
```

静默写入，不向用户展示。

---

### Step 6: 明日承接

基于今天的内容，简短列出明日可能的优先事项：

- 未完成但有动力的工作
- 近期承诺和截止日期
- 需要保持的活跃线索

展示格式：

```
明日承接：
- XXX（未完成，有动力）
- YYY（承诺，截止 MM-DD）
```

如果没有明确的承接事项，跳过此步。

---

### Quick Wrap（备选）

如果今日日志为空或内容很少，跳过 Step 1.5-4，直接用 AskUserQuestion 问 4 个快速问题：

1. 今天探索了什么新东西？
2. 什么事有进展？
3. 遇到什么卡点？
4. 明天最想做的一件事？

将回答写入日志和 AI 记忆文件。

---

## 注意事项

1. **速度优先**：5 分钟内完成，不做深度分析
2. **提取是轻量的**：只提取明显的条目，不做过度解读。深度提取留给 `日常-日志想法提取`
3. **联系发现是亮点**：通过 CLI 的 backlinks、search、orphans 发现的联系是本 skill 独特的价值
4. **文件创建遵循命名规范**：`YYYY-MM-DD（周几）.md`，星期用中文全称
5. **归档建议需确认**：不要未经确认就移动或创建文件
