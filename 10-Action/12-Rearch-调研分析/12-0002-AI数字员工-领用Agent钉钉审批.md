## 背景 

公司出台了 AI 数字人管理办法，里面要求领用 AI 数字人要经过部门长审批，并且记录是谁创建了数字人。 


## 任务内容 

调研领用 Agent 的审批流程设计 
明确部门长审批节点和权限 
设计数字人创建人记录机制 
输出产品方案并进行评审 


##  交付物 
- 审批流程设计文档
-  - 产品方案 PRD 
- - 评审会议纪要

## 实施方案

具体实施步骤：
第一步，完善钉钉审批流程；
第二步，打通钉钉接口的对接，审批数据落库。

因涉及到与调用钉钉接口的技术开发；
具体实施方案建议分两步走：

**第一阶段 钉钉审批+线下沟通**

完成第一步，所有的AI数字员工完成登记并分配AI数字人ID；

AI数字人中台保持现状，通过线下沟通方式来跟进。对于没有审批权限的AI数字人告知尽快完成审批，三天内不完成，将通过后台暂停AI数字员工使用。

**第二阶段 钉钉审批+线上验证**

完成与钉钉审批流程打通，实现线上验证并获取审批信息；

## 附件

### 钉钉审批调研

Q1: 调用钉钉审批结果及审批的日志数据，目前支持么？  
A: 支持审批结果查询，但审批日志数据部分支持。  
具体来说：审批实例详情（detail）、审批操作记录（records）、已发起的审批列表（list-initiated）、待处理列表（list-pending）这几类都支持，通过 `dws oa approval` 子命令调用。但"审批日志"如果指的是 list-executed（已处理）、list-submitted（已提交）、list-cc（抄送）这些，目前未实现，需要去钉钉 App 里查看。审批操作历史（即每条审批的流转记录）走 `dws oa approval records --instance-id <id>`，这个是支持的。

Q2: 是否可以获取审批内容  
A: 可以，通过钉钉开放平台的审批 API 能获取审批实例的表单内容。项目里 internal/apiclient/ 负责 HTTP 调用，你可以用 `dws` skill 里的审批能力来查询审批实例详情，返回数据中包含 `formComponentValues` 字段，里面就是每个表单项的填写内容（控件名、值、类型等）。需要先有审批实例 ID（processInstanceId）才能查。

Q3: 是否可以获取审批人填写内容

A: 这个项目本身是 MCP CLI 框架，不包含钉钉审批的业务逻辑，但可以通过 dws skill 调用钉钉审批接口。根据钉钉开放平台：  
审批实例详情接口（`processinstance.get`）返回的 `tasks` 字段里，每条审批任务包含 `result`（同意/拒绝）和 `remark`（审批意见文本）。审批人填写的表单控件内容在 `formComponentValues` 字段，格式是 JSON 数组，每项有 `name`（控件名）、`value`（填写值）、`extValue`（扩展值）。可以通过 `/dws` skill 调用 `dingtalk.oapi.processinstance.get` 直接拿到这些数据，`remark` 是纯文本，`formComponentValues` 是结构化 JSON。
