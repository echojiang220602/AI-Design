# 工单流转

> 工单系统验收标准与使用规范

## 验收工单系统的标准

### 1. 基础能力验收

| 验收项 | 标准 |
|--------|------|
| **工单创建** | 支持必填字段（标题、描述、优先级、类型）+ 可选字段（附件、抄送人） |
| **工单流转** | 支持状态流转：待处理 → 处理中 → 待验收 → 已完成 / 已关闭 |
| **工单分配** | 支持分配给指定处理人，支持转派 |
| **工单查询** | 支持按状态、创建人、处理人、日期范围筛选 |

### 2. 流程规范验收

| 验收项 | 标准 |
|--------|------|
| **状态变更记录** | 每一次状态变更都记录操作人、操作时间、变更原因 |
| **通知机制** | 状态变更时自动通知相关人（创建人 + 处理人） |
| **权限控制** | 创建者可关闭工单；处理人可更新状态；管理员可删除 |
| **验收节点** | 「待验收」状态需处理人主动提交验收申请，不能自动流转 |

### 3. 数据质量验收

| 验收项 | 标准 |
|--------|------|
| **字段完整性** | 必填字段未填写时阻止提交，提示明确 |
| **操作日志** | 所有操作记录不可删除，支持追溯 |
| **数据导出** | 支持按时间范围、工单类型导出 Excel |

### 4. 性能验收

| 验收项 | 标准 |
|--------|------|
| **响应时间** | 列表查询 < 1s，工单详情 < 500ms |
| **并发支持** | 支持 50+ 用户同时在线操作 |
| **数据容量** | 单工单附件 ≤ 50MB，历史数据保留 ≥ 1 年 |

## 工单类型定义

| 类型 | 适用场景 |
|------|----------|
| **Bug** | 缺陷报告 |
| **需求** | 功能需求 |
| **优化** | 体验优化、技术改进 |
| **咨询** | 疑问解答、操作指导 |
| **其他** | 不属于以上类型 |

---

## 优先级定义

| 优先级 | 说明 | 响应时间 |
|--------|------|----------|
| P0 | 紧急：系统宕机、业务中断 | 15 分钟 |
| P1 | 高：影响核心功能使用 | 2 小时 |
| P2 | 中：影响部分功能，非核心 | 1 工作日 |
| P3 | 低：无明显影响，可计划修复 | 3 工作日 |

---

## 安全验收

| 验收项 | 标准 |
|--------|------|
| **身份认证** | 支持 SSO/OAuth 登录，支持多因素认证（MFA） |
| **权限管理** | 基于角色的权限控制（RBAC），细粒度到操作级别 |
| **数据安全** | 敏感字段（手机号、身份证等）脱敏显示；传输全程 HTTPS |
| **审计日志** | 记录所有敏感操作（登录、删除、导出），日志保留 ≥ 180 天 |
| **数据隔离** | 不同租户/部门数据严格隔离，禁止越权访问 |

---

## 可用性验收

| 验收项 | 标准 |
|--------|------|
| **界面易用性** | 常用操作路径 ≤ 3 步，关键按钮可见可及 |
| **操作反馈** | 所有操作有明确状态提示（成功/失败/加载中） |
| **表单校验** | 实时校验 + 提交前校验，错误信息指向明确字段 |
| **移动端适配** | 核心流程支持移动端操作，响应式布局 |
| **帮助文档** | 每个功能模块有对应操作指引或 FAQ |

---

## 接口验收

| 验收项 | 标准 |
|--------|------|
| **API 规范** | RESTful 风格，响应格式统一（JSON），状态码语义明确 |
| **接口文档** | 使用 Swagger/OpenAPI 3.0 规范，示例完整 |
| **接口安全** | 关键接口需签名验证或 Token 校验，防重放攻击 |
| **接口限流** | 支持按用户/IP/接口维度的限流策略，429 响应合理 |
| **接口版本** | 支持版本号管理（如 /v1/, /v2/），老版本兼容期内可调用 |
