# SalesVoice2Note · 销售语音转结构化笔记

> 产品定位：AI 驱动的销售访谈记录工具，将录音自动转化为结构化问卷笔记

---

## 一、系统概述

### 核心价值

将销售/业务人员与客户的访谈录音，通过 ASR 语音识别 + LLM 智能解析，自动填充到预设问卷模板中，生成结构化笔记。

### 用户角色

- **销售主管**：陈喆榕（演示账号）
- **使用场景**：客户访谈、需求调研、产品演示、商务洽谈

### 产品类型

单页应用（SPA），纯前端实现，无需后端服务器即可运行。

## 六、关键设计模式

### 6.1 问卷复用设计（F-002/F-007）

模板预览和笔记渲染复用同一组件 `renderQuestionnaire()`，通过 `mode` 参数区分：

```javascript
// 模板预览模式（可编辑）
renderQuestionnaire(structure, {}, 'preview')

// 笔记填写模式（显示置信度）
renderQuestionnaire(structure, fieldValues, 'note')
```

### 6.2 归档硬约束（M-02）

```javascript
function noteArchive() {
  const redN = getRedFieldsCount();
  if (redN > 0) {
    toast(`🚫 存在 ${redN} 个红色低置信度字段未确认，无法归档`);
    return;
  }
  // 执行归档...
}
```

### 6.3 长文本编辑隔离（F-015）

编辑转写文本不影响字段值的设计：

```javascript
function transcriptEdit() {
  // 只更新转写文本
  noteTranscriptText = ta.value;
  // 不触发：noteFieldEdit() 或 LLM 重填
  // 状态保持：confirmed 字段不被修改
}
```

### 6.4 多轮对话状态管理

```javascript
// 状态栈
let snapshots = [];      // 历史快照（undo）
let redoStack = [];     // 重做栈
let currentStructure;   // 当前模板结构

// 每次变更前压栈
pushSnapshot();
currentStructure = newStructure;

// 撤销：恢复上一快照
const prev = snapshots.pop();
currentStructure = prev;
```

## 七、与通话记录生成的关系

### 功能对照

| 功能 | SV声析 | 通话记录生成 |
|------|--------|-------------|
| **输入** | 销售访谈录音 | 客户调研录音 |
| **模板** | 可自定义问卷（对话式） | 固定模板 |
| **字段提取** | LLM + 置信度 + 三色标识 | LLM |
| **确认机制** | 置信度分级 + 归档约束 | 手动修改 |
| **客户档案** | 关联拜访记录 | 未打通 |
| **长文本编辑** | 支持，不影响字段 | - |
| **音频播放** | 支持 | - |

### 可借鉴点

| 借鉴点 | 价值 |
|--------|------|
| 置信度系统 | 让 AI 填充质量一目了然，减少人工核对负担 |
