## 会议背景

为解决销售录音转化为结构化数据效率低、模板不可自定义等痛点，陈喆榕团队联合江灿设计“生息”产品方案，并与赵辉、刘美就温州港集团项目中的实际需求进行对齐。会上演示了语音转结构化模板、客户档案管理、AI填充与人工确认机制等功能，确认该方案可有效支持销售纪要自动化生成，并围绕工单派发、货源一张图联动、客户标签细化等业务场景提出优化方向。

## 生息产品方案演示与反馈

- 功能架构
    
    - 模板中心
        
        - 支持通过自然语言对话方式创建或修改模板，实现非技术人员自定义结构化字段
            
        - 模板最终映射为 KV 型 JSON 结构，支持字符串、数字等类型约束
            
        - 使用 AI 图 EY 协议将对话内容自动转化为前端问卷式模板
            
    - 录音解析流程
        
        - 用户上传录音文件后，系统自动解析原文并由 AI 填充至指定模板
            
        - 提供 AI 填充自信度提示，低置信度项需人工复核
            
        - 支持追加后续拜访录音，持续累积客户维度笔记
            
    - 输出与归档
        
        - 支持导出 HTML 格式纪要，便于分享给上级或协作方
            
        - 可对历史客户数据执行归档操作，形成 CMC 管理闭环
            
    - 客户档案模块
        
        - 展示客户基本信息、所属行业、联系人等静态信息
            
        - 按时间线展示历次访谈记录，支持横向对比不同阶段客户状态
            
        - 计划新增里程碑事件记录（如成交、引入、复购），用于追踪转化路径
            
- 赵辉反馈
    
    - 整体评价：demo 已较完整覆盖上次会议提出的痛点，尤其在模板自定义和客户档案建设方面有显著提升
        
    - 补充建议
        
        - 需增加工单关联板块，将走访任务以工单形式嵌入客户档案，体现任务派发与执行过程
            
        - 客户档案中应支持记录多维度属性，如同名客户但不同地址/公司/客户等级的情况
            
        - 建议在历史访谈页增加里程碑标记功能，用于标注关键业务进展（如首次成交、二次转化）
            
- 江灿回应
    
    - 同意在客户档案中增加“客户来源”字段，支持从“货源一张图”跳转链接，实现双向打通
        
    - 可在历史访谈区域增加“重点事件”条目，支持手动录入或 AI 自动识别关键节点
        
    - 工单模块可作为独立功能嵌入当前系统，后续通过接口对接实现数据同步
        

## 工单系统现状与集成方案讨论

- 现有工单流程（赵辉介绍）
    
    - 当前系统为本地化部署的“温州港物流一体化”平台
        
    - 流程包括：新增工单 → 选择类型（如易仿工单）→ 关联项目与企业 → 设置默认走访日期 → 状态为“待执行”
        
    - 执行机制：
        
        - 若指派固定人员，则仅该人员可见；若未指定，则开放“抢单”模式
            
        - 开始执行后进入线下走访，遇问题可发起“协助请求”，流转至管理层处理
            
        - 处理完成后可继续流转回执行人，或终止任务形成闭环
            
- 集成模式探讨
    
    - 赵辉提议：希望将“生息”系统的功能模块以接口形式嵌入现有系统，避免重复建设
        
    - 技术可行性评估
        
        - 陈喆榕指出两系统前端框架可能不一致（温州系统用 VUE，生息系统拟采用流式渲染框架），直接嵌套存在技术障碍
            
        - 建议采用页面跳转+参数传递方式实现联动，确保用户身份、客户信息等上下文完整带入
            
    - 刘美建议
        
        - 杭州侧可构建通用能力平台，提供标准化接口服务
            
        - 客户端展示层仍由各系统自行实现，以满足个性化 UI 需求
            
        - “大而全”的后台能力适用于内部提效，对外交付时可根据客户需要裁剪功能集
            

## 货源地分析精准度优化需求

- 当前痛点
    
    - 当前通过集卡定位反推企业坐标的算法存在匹配不准问题
        
    - 典型场景：
        
        - 园区密集型企业无法精确区分具体归属
            
        - 同一栋楼多个企业共用坐标（如七层每层一家公司）
            
        - 大厂区多出入口导致定位漂移（如东门靠近邻近小作坊）
            
    - 客户期望：能精准识别到门牌号级别，减少人工判断成本
        
- 改进方向
    
    - 陈喆榕表示其团队拥有较强算法资源，可协助优化定位模型
        
    - 赵辉补充希望进一步实现货源流向可视化：
        
        - 识别出的企业与其出运航线数据挂钩
            
        - 构建完整链路：货源地 → 装箱点 → 出运码头 → 干线/支线船信息
            
    - 数据范围：目前关注中国出口港即可，暂无需延伸至海外港口
        
- 业务价值
    
    - 帮助客户掌握区域内货源分布全貌，弥补数据缺失
        
    - 辅助航线开发决策：基于企业出口国家偏好定向排船，提升运营效率

###### AI洞察

- 本次会议体现出典型“业务痛点驱动+AI能力反哺”的创新模式：一线业务提炼真实场景（录音转纪要、工单闭环、货源可视），总部技术团队快速响应并封装成可复用平台。
    
- 客户真正诉求并非工具本身，而是“零脑力消耗的结果输出”——无论是精准到门牌号的定位，还是自动生成的销售纪要，本质都是追求决策确定性。
    
- 跨系统集成面临“技术异构”与“组织边界”双重挑战，采用“能力解耦+界面分离”策略（通用引擎+本地化UI）或是破解之道。

# 章节

- 00:00:00 会议开场与需求背景介绍
    
    - 会议由陈喆榕开场，介绍了当前业务中录音转结构化数据的需求背景。他提到上次沟通后已形成初步方案，并指出目前结构化数据依赖前端开发导致周期较长的问题。本次会议旨在讨论新提出的产品方案是否真正解决核心痛点。
        
- 00:01:07 产品方案演示：语音转结构化模板
    
    - 江灿开始演示名为“生息”的产品方案，展示如何通过自然语言对话方式生成可自定义的结构化输入模板。该模板能将语音内容转化为JSON格式的数据结构，并支持AI填充与人工确认机制，提升信息提取效率。
        
- 00:02:10 技术实现细节与交互逻辑说明
    
    - 陈喆榕进一步解释产品背后的AI技术路径，说明左侧对话可转化为右侧问卷式模板，利用AI to EY协议实现自然语言到结构化Schema的转换。系统通过设定字段类型约束（如字符串、数字）确保数据规范性。
        
- 00:05:37 功能模块详解：模板管理与客户档案
    
    - 江灿详细介绍产品功能，包括模板自定义、录音上传解析、AI填充结果展示、HTML导出、客户档案管理及归档功能。系统支持按时间线查看访谈记录，并提供横向对比维度，构建小型CRM闭环。
        
- 00:10:50 用户反馈与功能补充建议
    
    - 赵辉对demo表示认可，认为已覆盖此前讨论的痛点。同时提出三项关键补充需求：增加工单派发模块以对接走访任务；建立与‘货源一张图’系统的双向链接；在客户档案中加入里程碑事件追踪，如成交转化等关键节点。
        
- 00:14:35 系统对接与数据联动方案探讨
    
    - 针对赵辉提出的‘货源一张图’需求，江灿建议在客户档案中添加来源标签并实现双链跳转。双方讨论多地址同名客户的区分方式，决定引入公司名称、客户等级、标签等属性进行精准识别。
        
- 00:19:42 合作模式与项目推进机制确认
    
    - 陈喆榕提出两种合作模式：由赵辉直接向江灿提需求或双方团队协作开发。赵辉因项目核算和节奏控制问题，倾向于前者。陈喆榕承诺承担成本，并强调赵辉和刘美作为需求发起方的核心地位。
        
- 00:24:40 应用场景扩展与内外部使用讨论
    
    - 团队探讨该产品的适用范围，确认其最初源于温州港集团揽货人员的实际痛点，未来可拓展至台州及其他业务场景。同时评估内部销售团队的应用可能性，如物流宝电销场景，推动平台通用化建设。
        
- 00:30:54 系统集成与技术对接方案分析
    
    - 针对不同技术栈（Vue vs 新框架）带来的嵌套难题，团队讨论采用页面跳转+参数传递的方式实现系统间协同。刘美建议提供通用接口支撑功能实现，前端展示则保留各系统独立风格，兼顾灵活性与一致性。
        
- 00:37:04 核心痛点深化：货源地精准识别与流向分析
    
    - 赵辉指出当前货源地识别存在坐标匹配不精确的问题，难以定位到具体楼层或厂区门牌。团队提出需结合GIS与算法优化，目标是实现从集卡数据到企业精确位置的高精度映射，并关联航线出运数据，构建完整货源流向链路。
        
- 00:46:52 会议总结与后续行动计划
    
    - 会议尾声，各方确认后续动作：江灿将整理项目信息发送赵辉；工时填报统一归入AI数字人中台；需求优先级调整为最高。陈喆榕强调该工作已获高层支持，将定期向于波总与叶董事长汇报进展。
