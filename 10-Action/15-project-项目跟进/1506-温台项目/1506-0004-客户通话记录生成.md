# 客户通话记录生成需求

---

## 一、需求背景（300字）

### 业务背景

在港口物流及货运代理业务中，一线业务人员需定期拜访客户进行需求调研，采集企业联系人、主要产品、出运方式、贸易条款等关键信息。传统手工记录方式存在以下痛点：

**效率低下**：每次拜访后需手动整理录音内容，耗时长，影响业务人员外出时间。

**信息遗漏**：依赖人工记忆难免遗漏关键信息，且不同人员记录格式不统一，后期难以汇总分析。

**数据孤岛**：调研数据独立存储，未与企业客户画像体系打通，无法支撑精准营销决策。

### 解决方案

通过实现**录音采集→ASR语音转写→LLM智能提取→结构化报告**的自动化流程，将业务人员从繁重的手工记录中解放出来。调研数据自动入库，后续可与AI数字人中台知识库打通，形成统一的企业客户画像体系。

### 核心价值

- **提升调研效率**：自动化生成结构化报告，释放业务人员精力
- **信息完整准确**：AI智能提取，减少人工遗漏
- **数据资产化**：调研数据统一沉淀，支持后续分析与应用

## 二、需求描述

### 2.1 第一阶段：录音转报告（已完成基础版）

#### 功能需求

| 序号 | 功能点 | 描述 | 优先级 |
|------|--------|------|--------|
| F1.1 | 录音采集 | 一线人员使用录音笔采集客户访谈音频 | P0 |
| F1.2 | 文件上传 | 通过系统附件入口上传录音，后台异步解析 | P0 |
| F1.3 | ASR转写 | 语音识别模型将录音转为文本，全量转写 | P0 |
| F1.4 | 对话分离 | 区分客户与业务人员的对话内容 | P1 |
| F1.5 | 字段提取 | LLM解析文本，提取联系人、产品、出运方式等关键字段 | P0 |
| F1.6 | 报告生成 | JSON数据渲染至模板，生成调研报告 | P0 |
| F1.7 | 手动修正 | 支持用户对生成内容调整后保存 | P1 |

#### 数据需求

| 数据源 | 数据内容 | 提供方 |
|--------|----------|--------|
| 录音文件 | 客户访谈原始音频 | 一线业务人员（录音笔） |
| 模板配置 | 调研报告字段模板 | 业务团队 |

#### 技术需求

| 需求 | 描述 |
|------|------|
| T1.1 | 录音文件上传与存储模块 |
| T1.2 | ASR语音识别（方言语料优化） |
| T1.3 | 对话分离与语义理解模块 |
