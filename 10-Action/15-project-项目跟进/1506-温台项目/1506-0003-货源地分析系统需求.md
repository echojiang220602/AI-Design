# 货源地分析系统需求

> 版本：v1.0
> 日期：2026-06-29
> 依据：0625温台项目会议纪要

---

## 一、需求背景（300字）

### 业务背景

随着港口物流行业竞争加剧，精准识别货源地成为提升业务效率的关键。当前物流企业面临以下痛点：

**数据缺失**：温州地区缺乏完善的工厂地址库，业务团队难以精准定位潜在客户，只能依赖传统人工拜访方式，效率低下。

**线索分散**：集卡车辆的运输轨迹中蕴含大量有价值的货源信息，但未经系统化分析，无法形成有效的商业线索。

**覆盖有限**：现有货源分析仅覆盖浙南区域，缺乏全省乃至跨区域的整体视图，无法支撑全局业务决策。

### 解决方案

通过整合港口进出港报文数据、交通厅车辆轨迹数据，结合AI算法模型，自动识别可疑工厂停留点，在地图上可视化呈现，辅助业务团队快速定位潜在客户，提升线下拜访效率。

### 核心价值

- **降低获客成本**：通过数据驱动精准定位，减少无效拜访
- **提升业务效率**：AI辅助分析，释放人力资源
- **全省一张网**：打破区域壁垒，实现货源全链路追踪

## 二、需求描述

### 2.1 第一阶段：货源地初步分析（已完成）

#### 功能需求

| 序号 | 功能点 | 描述 | 优先级 |
|------|--------|------|--------|
| F1.1 | 报文数据接入 | 接入浙南区集卡车辆进出港报文，包含车牌号、进出时间 | P0 |
| F1.2 | 轨迹数据调用 | 调用交通厅接口，获取车辆轨迹数据 | P0 |
| F1.3 | 停留点识别 | 基于停留时长（60分钟、120分钟）和距离半径（100米、500米、1公里）两个维度，识别可疑工厂停留点 | P0 |
| F1.4 | 地图可视化 | 蓝点标注可疑工厂，红点显示已入库企业（温州本地1万+家） | P0 |
| F1.5 | 区域查询 | 支持浙江省温州市平阳县等区域筛选 | P1 |
| F1.6 | 时间范围选择 | 支持2015.10.1-2016.1.24 或 2025.10.1至今 | P1 |
| F1.7 | 企业更正 | 线下拜访确认后，对接企查查将蓝点转化为准确点 | P2 |

#### 数据需求

| 数据源 | 数据内容 | 提供方 |
|--------|----------|--------|
| 进出港报文 | 车牌号、进出港时间 | 港口系统 |
| 车辆轨迹 | 车辆行驶轨迹数据 | 交通厅接口 |
| 企业库 | 温州已入库企业1万+ | 会员系统 |

#### 技术需求

| 需求 | 描述 |
|------|------|
| T1.1 | 报文解析模块 |
| T1.2 | 交通厅接口调用与数据对接 |
| T1.3 | 停留点识别算法（时间+距离双维度） |
| T1.4 | 地图可视化组件 |
| T1.5 | 企查查API对接（企业信息查询） |
