## 三、命令详解

### git status
查看当前工作目录的改动状态：
- `M` = 已修改（modified）
- `D` = 已删除（deleted）
- `??` = 未跟踪的新文件（untracked）
- `A` = 已新增并暂存（added）

### git add -A
将所有改动加入暂存区，涵盖：
- 所有已修改的文件
- 所有已删除的文件
- 所有新增的未跟踪文件

**注意**：已被 `.gitignore` 规则忽略的文件不会被加入。

### git commit -m "..."
将暂存区的改动写入本地仓库历史，生成一个 commit。

规范建议：
- 第一行：简短标题（不超过 50 字）
- 空一行
- 后续：详细说明改动了什么

示例：
```bash
git commit -m "feat: 技能列表优化 & 技能详情页背景色调整

- 技能工具列表改名为技能列表，仅保留技能数据
- 移除技能项的 Switch 开关控件
- 新增 7 个技能（doc-writer, image-generator 等共 10 个）
- 技能详情页：技能信息条和正文内容区改为浅蓝背景 #F0F9FF"
```

### git push origin main
将本地 main 分支推送到远程 origin（GitLab）。

输出示例（成功时）：
```
To https://gitlab.4portun.com/booking-bailout/clawmanager-prd.git
   8a8d7bca..015172aa  main -> main
```

## 四、常见场景

### 场景1：仅提交部分文件
如果只想提交部分文件，而不是 `git add -A`：

```bash
git add 文件路径1 文件路径2
git commit -m "描述"
git push origin main
```

### 场景2：撤销 git add（还没 commit）
```bash
git reset HEAD
```

### 场景3：撤销已 commit 但还没 push
```bash
git reset --soft HEAD~1   # 保留改动在暂存区
# 或
git reset --mixed HEAD~1  # 保留改动在工作区（默认）
# 或
git reset --hard HEAD~1   # 彻底丢弃改动（危险）
```

### 场景4：远程有更新，本地落后
```bash
git pull --rebase origin main
git push origin main
```

### 场景5：查看提交历史
```bash
git log --oneline -10
```

## 六、注意事项

1. **commit 之前先看 git status**：确认改动了什么，避免误提交不需要的文件
2. **commit message 要清晰**：方便日后回溯
3. **push 之前确认分支**：当前仓库默认分支是 `main`，不要推错分支
4. **大文件不要进仓库**：如 node_modules、dist、.vite 等已被 .gitignore 排除
5. **敏感信息不要进仓库**：如 .env 文件（包含密钥、密码等）
