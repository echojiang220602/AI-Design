## 一、需求概述

### 1.1 建设目标

将现有「技能商店」升级为完整的**技能商城**，提供技能「发现 → 使用 → 发布 → 管理」全链路能力，形成用户贡献与平台能力的正向循环。

### 1.2 现有能力 vs 目标能力

| 维度 | 现有能力 | 目标能力 |
|------|---------|---------|
| 技能展示 | 技能包 + 全部技能列表 | 技能市场（四港精选）+ 搜索发现 |
| 筛选维度 | 按标签筛选（官方/热门/必装） | 按分类 + 场景 + 排序多维筛选 |
| 搜索能力 | 无 | 关键词模糊搜索 |
| 技能来源 | 平台内置 | 平台内置 + 用户贡献 |
| 发布流程 | 无 | 用户提交 → 安全扫描 → 审批 → 上架 |
| 运营能力 | 无 | 推荐位、置顶、分类配置 |

### 1.3 页面结构

```
左侧导航 → 技能商城（默认 Tab：四港精选）
    │
    ├── 【Tab2】四港精选  ← 默认展示页
    │      ├── 分类筛选（全部/按分类）
    │      ├── 排序切换（综合/热度/最新/评分）
    │      ├── 置顶推荐位
    │      └── 技能卡片网格
    │
    ├── 【Tab1】技能市场
    │      ├── 搜索框（关键词搜索）
    │      ├── 搜索结果列表
    │      ├── 热门推荐
    │      └── 搜索历史
    │
    └── 【Tab3】发布 Skill
           ├── 提交表单
           ├── 状态时间线
           └── 扫描结果展示

管理员菜单 → 审批管理（独立页面）
    ├── 审批列表
    └── 审批详情 + 操作
```

### 2.3 发布 Skill Tab

#### 定位
用户提交技能申请，经过安全扫描和审批后上架商城。

#### 提交流程

```
步骤1：填写信息
    ├── 技能名称（必填，最长 30 字）
    ├── 分类（必填，下拉选择）
    ├── 一句话简介（必填，最长 50 字）
    ├── 核心功能说明（必填，最长 200 字）
    ├── 使用场景（必填，最长 200 字）
    └── 示例提示词（必填，20-2000 字，支持代码块）

步骤2：提交申请
    ↓
步骤3：安全扫描（Cisco SafeX）
    ├─ 敏感词检测
    ├─ 恶意代码检测（正则引擎）
    └─ 提示词注入检测（规则 + LLM）
    ├─ 扫描通过 → 进入「待审批」队列
    └─ 扫描失败 → 展示失败原因，可修改重提

步骤4：管理员审批
    ├─ 通过 → 自动上架「四港精选」
    └─ 拒绝 → 附原因，可修改重提

步骤5：上架完成（全量用户可见）
```

#### 状态定义

| 状态 | 标签颜色 | 用户可见 |
|------|---------|---------|
| `PENDING_SCAN` | 蓝 | 提交成功，等待扫描 |
| `SCANNING` | 蓝 | 扫描进行中 |
| `SCAN_PASS` | 绿 | 扫描通过，等待审批 |
| `SCAN_FAIL` | 红 | 扫描失败，查看原因后可重提 |
| `PENDING_APPROVAL` | 蓝 | 等待审批 |
| `APPROVED` | 绿 | 审批通过 |
| `REJECTED` | 红 | 审批拒绝，可重提 |
| `PUBLISHED` | 绿 | 已上架（全量可见） |

#### 表单字段

| 字段 | 类型 | 必填 | 限制 |
|------|------|------|------|
| 技能名称 | 文本 | 是 | 最长 30 字 |
| 分类 | 下拉 | 是 | 按商城分类体系 |
| 一句话简介 | 多行文本 | 是 | 最长 50 字 |
| 核心功能说明 | 多行文本 | 是 | 最长 200 字 |
| 使用场景 | 多行文本 | 是 | 最长 200 字 |
| 示例提示词 | 多行文本 | 是 | 20-2000 字，支持代码块 |

### 3.5 搜索技术方案

**推荐：PostgreSQL 全文搜索 + pg_trgm**（零新增基础设施）

```sql
-- 新增搜索索引（基于现有 skills 表扩展）
ALTER TABLE skills ADD COLUMN name_tsv TSVECTOR;
ALTER TABLE skills ADD COLUMN desc_tsv TSVECTOR;
ALTER TABLE skills ADD COLUMN tags_tsv TSVECTOR;

-- GIN 索引
CREATE INDEX idx_skills_name_tsv ON skills USING GIN (name_tsv);
CREATE INDEX idx_skills_desc_tsv ON skills USING GIN (desc_tsv);

-- trigram 模糊搜索索引
CREATE EXTENSION IF NOT EXISTS pg_trgm;
CREATE INDEX idx_skills_name_trgm ON skills USING GIN (skill_name gin_trgm_ops);
```

搜索 API 实现：PostgreSQL `to_tsvector` + `ts_query` 全文检索，结合 `similarity()` trigram 模糊匹配。

### 3.6 安全扫描集成

**接入 Cisco SafeX API**（开源，有 API）

```typescript
// services/scanService.ts
const SAFEX_API_URL = import.meta.env.VITE_SAFEX_API_URL;
const SAFEX_API_KEY = import.meta.env.VITE_SAFEX_API_KEY;

async function callSafeX(text: string) {
  const res = await fetch(`${SAFEX_API_URL}/api/v1/check`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${SAFEX_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      text,
      categories: ['politics', 'violence', 'porn', 'ad', 'fraud'],
    }),
  });
  return res.json();
}
```

## 六、关键技术决策

### 6.1 搜索 vs 技能包分类的关系

| 问题 | 决策 |
|------|------|
| 四港精选的分类沿用技能包体系吗？ | **是**，复用现有的 6 个技能包分类（工程师/PM/运维/行政/HR/商务），新增 `custom` 分类给用户自建技能 |
| 搜索结果里的技能和四港精选技能是同一套数据吗？ | **是**，底层都是 `skills` 表，`is_published = true` 即上架，`is_pinned = true` 即置顶 |
| 技能包还要保留吗？ | **保留**，技能包是面向用户的快捷安装入口，与四港精选并行存在 |

### 6.2 技能来源与可见性

| 技能来源 | 是否上架 | 可见范围 |
|---------|---------|---------|
| 平台内置（official） | 自动上架 | 全量用户 |
| 用户提交（Cisco SafeX 扫描通过 + 审批通过） | 审批通过后自动上架 | 全量用户 |
| 用户提交（扫描失败/审批拒绝） | 不上架 | 仅申请人可见 |

### 6.3 与 SkillHub 的关系

| 场景 | 关系 |
|------|------|
| 技能市场搜索 | 从 SkillHub 拉取全量技能数据，同步入搜索索引 |
| 发布 Skill | 提交后直接进入审批流，不走 SkillHub |
| 上架后 | 技能同时出现在四港精选和 SkillHub |
