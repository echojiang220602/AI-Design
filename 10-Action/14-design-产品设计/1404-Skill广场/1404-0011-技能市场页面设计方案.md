# 「技能商城」页面整体设计方案

> 本文档是「技能商城」页面从 0 到 1 的完整设计方案，基于截图分析（搜索列表页 + Skill 详情页）输出，包含产品结构、交互逻辑、视觉规范、技术实现和数据源说明。可作为 UI/UX 设计交付和前端开发规格的双向依据。

---

## 一、产品定位与用户场景

**用户**：希望发现、安装和管理 AI 技能/工具的终端用户

**核心场景**：
1. 浏览技能商城，发现感兴趣的技能或工具
2. 通过关键词搜索快速定位目标技能
3. 按类型筛选（全部 / 技能 / 工具）
4. 点击卡片查看技能详情（功能介绍、使用说明、更新日志）
5. 安装或卸载技能到自己的 Agent

**设计原则**：
- 信息密度适中：卡片列表兼顾预览效率与信息完整度
- 视觉引导清晰：分类标签色系区分技能（绿）与工具（青）
- 操作即时反馈：安装按钮状态切换流畅

## 三、技能商城列表页（图一）详细设计

### 3.1 布局结构

```
┌────────────────────────────────────────────────────────────────┐
│  侧边导航栏（展开态，固定宽度 220px）                            │
│  ├─ 首页                                                        │
│  ├─ 技能商城（当前激活，蓝色高亮）                                │
│  ├─ 我的智能体                                                  │
│  └─ 文件夹                                                      │
├────────────────────────────────────────────────────────────────┤
│  主内容区（flex-1，overflow-y: auto）                           │
│  ├─ 顶部栏（搜索区 + Tab 切换）                                 │
│  │   ├─ 页面标题 "技能商城"                                      │
│  │   ├─ 搜索输入框（右侧）                                       │
│  │   └─ Tab 标签：全部 / 技能 / 工具                             │
│  └─ 卡片网格（2 列，gap-4）                                     │
└────────────────────────────────────────────────────────────────┘
```

**技术实现**：
- 容器：`flex h-screen`（左侧 sidebar + 右侧主内容）
- 主内容区：`flex-1 overflow-y-auto p-6`
- 网格：`grid grid-cols-2 gap-4`

### 3.2 搜索与筛选

#### 3.2.1 搜索输入框

| 元素 | 规格 |
|---|---|
| 宽度 | `w-64` |
| 高度 | `h-10`（40px） |
| 圆角 | `rounded-lg`（8px） |
| 边框 | `border border-neutral-200` |
| 占位符 | `"搜索技能或工具..."` |
| 左侧图标 | `Search`，16px，`text-neutral-400` |
| 聚焦态 | `border-[#0099DD]`，`ring-2 ring-[rgba(0,153,221,0.2)]` |

**实现代码**：
```tsx
<div className="relative">
  <Search className="absolute left-3 top-1/2 -translate-y-1/2 w-4 h-4 text-neutral-400" />
  <input
    type="text"
    placeholder="搜索技能或工具..."
    value={keyword}
    onChange={(e) => setKeyword(e.target.value)}
    className="w-64 h-10 pl-9 pr-4 rounded-lg border border-neutral-200
      text-sm text-neutral-800 placeholder:text-neutral-400
      focus:outline-none focus:border-[#0099DD] focus:ring-2
      focus:ring-[rgba(0,153,221,0.2)] transition-all"
  />
</div>
```

#### 3.2.2 Tab 切换

| Tab 标签 | 样式 |
|---|---|
| 全部 | 默认灰底灰字 `bg-neutral-100 text-neutral-500`；激活蓝底白字 `bg-[#0099DD] text-white` |
| 技能 | 同上 |
| 工具 | 同上 |

**样式规格**：
- Tab 容器：`flex gap-2 p-1 bg-neutral-100 rounded-lg`（背景 pill 形态）
- 单个 Tab：`px-4 py-1.5 rounded-md text-sm font-medium transition-all`
- 激活态：`bg-[#0099DD] text-white shadow-sm`
- 默认态：`text-neutral-500 hover:text-neutral-700`

### 3.3 技能卡片组件规格

#### 3.3.1 卡片整体规格

| 属性 | 值 |
|---|---|
| 背景 | `bg-white` |
| 圆角 | `rounded-xl`（12px） |

#### 3.3.2 技能图标

| 属性 | 值 |
|---|---|
| 容器尺寸 | `w-14 h-14`（56px） |
| 容器样式 | `rounded-2xl`，彩色渐变背景（每个技能独立配色） |
| 图标尺寸 | `w-7 h-7`（28px） |
| 图标对齐 | 居中 |
| 渐变色 | 示例：`bg-gradient-to-br from-blue-400 to-blue-600`，`from-green-400 to-green-600` 等 |

**渐变色规范（建议 8 套）**：

| 技能类型关键词 | 渐变方向 | 起始色 | 结束色 |
|---|---|---|---|
| 代码 / 开发 / coding | to-br | `#3B82F6` 蓝 | `#1D4ED8` 深蓝 |
| 数据 / database | to-br | `#8B5CF6` 紫 | `#6D28D9` 深紫 |
| 搜索 / search | to-br | `#06B6D4` 青 | `#0891B2` 深青 |
| 报告 / 总结 | to-br | `#F97316` 橙 | `#EA580C` 深橙 |
| 文件 / 文件处理 | to-br | `#22C55E` 绿 | `#16A34A` 深绿 |
| 安全 / 监控 | to-br | `#EF4444` 红 | `#DC2626` 深红 |
| 通用 / 默认 | to-br | `#0099DD` 蓝 | `#0077BB` 深蓝 |
| 其他 | to-br | `#6B7280` 灰 | `#4B5563` 深灰 |

#### 3.3.3 文字内容区

| 元素 | 规格 |
|---|---|
| 技能名称 | `text-base font-semibold text-neutral-800 mt-3`，单行，超长截断 `truncate` |
| 描述文字 | `text-sm text-neutral-500 mt-1`，最多 2 行，`line-clamp-2` |
| 行高 | 描述文字 `leading-relaxed`（1.625） |

#### 3.3.4 分类标签

| 类型 | 背景色 | 文字色 | 字号 |
|---|---|---|---|
| 技能（skill） | `bg-green-100` | `text-green-700` | `text-xs font-medium px-2 py-0.5 rounded-full` |
| 工具（tool） | `bg-cyan-100` | `text-cyan-700` | 同上 |

**实现代码**：
```tsx
<span className={`inline-block text-xs font-medium px-2 py-0.5 rounded-full
  ${type === 'skill'
    ? 'bg-green-100 text-green-700'
    : 'bg-cyan-100 text-cyan-700'
  }`}>
  {type === 'skill' ? '技能' : '工具'}
</span>
```

#### 3.3.5 底部指标区

| 元素 | 规格 |
|---|---|
| 安装数 | `Download` 图标 14px `text-neutral-400` + 数字 `text-sm text-neutral-500` |
| 评分 | 5 颗星，`Star` 图标 12px，填充色 `#F97316`（橙色），空心 `text-neutral-300` |
| 布局 | `flex items-center justify-between mt-3` |

**评分星实现**：
```tsx
<div className="flex items-center gap-0.5">
  {[1,2,3,4,5].map((i) => (
    <Star
      key={i}
      className={`w-3 h-3 ${i <= rating ? 'fill-[#F97316] text-[#F97316]' : 'text-neutral-300'}`}
    />
  ))}
</div>
```

### 3.4 卡片网格布局规范

| 属性 | 值 |
|---|---|
| 列数 | 2 列（`grid-cols-2`） |
| 列间距 | `gap-x-4`（16px） |
| 行间距 | `gap-y-4`（16px） |
| 最大宽度 | `max-w-[1200px] mx-auto` |
| 响应式（<768px） | `grid-cols-1` 单列 |

**每行卡片高度对齐策略**：同 1403 方案，JavaScript 高度同步（`ResizeObserver` + `requestAnimationFrame`）。

### 3.5 悬浮操作按钮

| 属性 | 值 |
|---|---|
| 位置 | `fixed bottom-6 right-6` |
| 尺寸 | `w-14 h-14`（56px 圆形） |
| 背景色 | `#F97316` 橙色 |
| 圆角 | `rounded-full` |

<div className="flex gap-6 border-b border-neutral-200 mb-6">

## 六、交互规范

### 6.1 搜索交互

- **触发**：用户输入文字时，实时过滤（`debounce 300ms`）
- **匹配**：模糊匹配技能名称 + 描述字段
- **空结果**：显示空状态 "未找到匹配 '{keyword}' 的技能或工具"，引导语 "试试其他关键词"
- **清空**：点击输入框右侧清除图标（`X`），清空后恢复全量列表

### 6.2 Tab 切换（全部 / 技能 / 工具）

- **行为**：立即切换，无动画，触发列表重新渲染
- **状态**：受控组件，`filterType: 'all' | 'skill' | 'tool'`
- **搜索 + Tab 组合**：Tab 筛选与搜索关键词同时生效（AND 逻辑）
- **URL 同步**（可选）：`?tab=skill&q=代码`（方便分享链接）

### 6.3 卡片点击

- **行为**：点击卡片跳转详情页 `router.push(\`/skill/${skill.id}\`)`
- **悬停态**：`shadow-md`，`border-[rgba(0,153,221,0.2)]`，`cursor-pointer`
- **点击态**：`scale-[0.98]`，150ms，`ease-out`

### 6.4 安装流程

```
用户点击"安装"按钮
  ↓
按钮文字变为"安装中..." + Spinner 图标（禁用点击）
  ↓
调用 POST /api/skills/{id}/install
  ↓
成功 → 按钮变为"已安装"（蓝底白字变白底蓝字过渡）
失败 → Toast 提示"安装失败，请重试"，按钮恢复"安装"
```

**安装按钮状态机**：

| 状态 | 文字 | 样式 | 可点击 |
|---|---|---|---|
| idle（未安装） | "安装" | 白底蓝字 | 是 |
| loading | "安装中..." | 白底蓝字 + Spinner | 否 |
| installed | "已安装" | 透明底白字 | 否 |
| error | "安装" | 白底蓝字 | 是 |

### 6.5 详情页 Tab 切换

- **行为**：点击立即切换，无动画
- **内容渲染**：根据 `activeTab` 渲染对应内容组件
- **键盘支持**：方向键 `←` `→` 切换（`tabindex` + `onKeyDown`）

## 七、技术实现摘要

### 7.1 技术栈

- **框架**：React 18 + TypeScript（严格模式）

  - 客户端全局状态：Zustand（当前筛选 Tab、搜索关键词）

### 7.3 状态管理

| 状态 | 工具 | 说明 |
|---|---|---|
| 技能列表数据 | `useQuery(['skills', filterType, keyword])` | TanStack Query |
| 当前 Tab | `useState<'all'\|'skill'\|'tool'>` | 组件内局部状态 |
| 搜索关键词 | `useState<string>('')` | 组件内局部状态（debounced） |
| 安装状态 | `useMutation` + `queryClient.invalidateQueries` | TanStack Query |

- 列表页：卡片列表使用 `windowing`（虚拟列表）优化大量数据场景

- 悬浮按钮：`React.lazy` 导入创建页面，减少首屏 JS 体积
