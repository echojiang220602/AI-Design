# Skill 技能市场 — 页面设计参考

> 来源：MiniMax M2.7 技能市场截图（搜索"PPT"）+ OpenClaw 设计文档

---

## 整体设计哲学

> 专业工具市场，而非消费级应用商店

这套界面的气质非常克制。它不像消费端的 App Store 那样依赖大图、评分、榜单来吸引眼球，而是以**信息密度和扫描效率**为第一优先级——每个技能条目在一行内同时传达名称、描述、分类、热度和来源，用户不需要点进去就能完成 80% 的筛选决策。这是面向有明确目标的专业用户的设计逻辑。

### 设计分析

#### 信息层级（F型视线扫描路径）

每行条目的视觉权重从左到右依次递减：

| 顺序 | 元素 | 作用 |
|------|------|------|
| 1 | 图标 | 建立品牌感 |
| 2 | 名称 + 描述 | 传递核心价值 |
| 3 | 分类标签 | 快速归类 |
| 4 | 数字指标（收藏/使用量） | 社交证明 |
| 5 | 来源标注 | 信任背书 |

#### 分类标签的颜色语义

| 分类 | 颜色 | 说明 |
|------|------|------|
| AI Agent | 蓝色 | 语义区分 |
| 办公效率 | 橙色 | 品牌色 |
| 设计多媒体 | 紫色 | 语义区分 |

> 不同类别有独立颜色，并非装饰性的，而是具有语义功能，让用户在快速扫列表时能用颜色而非文字来区分品类。这是一个成熟的信息设计决策。

#### 视图切换

右上角提供**列表 / 卡片**两种视图切换：
- 快速比较多个选项 → 用列表
- 深度了解单个技能 → 用卡片

> 这是对用户行为多样性的尊重，而不是强迫所有人走同一种路径。

#### 待优化：搜索词高亮缺失

截图中搜索「ppt」后，列表里的结果名称和描述中**并没有对匹配关键词做高亮处理**。这在技能数量较多时会让用户难以快速定位"为什么这个结果会出现"，尤其是描述文字被截断的情况下。

**建议**：搜索词高亮显示，增强匹配透明度。

## 页面二：技能详情页（Skill Detail）

### 截图参考

![[attachments/02_detail_1.webp|详情页-1]] ![[attachments/03_detail_2.webp|详情页-2]]

### 设计分析

#### 双栏布局的信息优先级

| 区域 | 内容 | 逻辑 |
|------|------|------|
| **左侧** | 内容主体（所有需要阅读的信息） | 先理解 |
| **右侧** | 行动区（安装/复制 prompt） | 再行动 |

> 这个布局传达了一个清晰的产品逻辑：先让用户理解，再引导用户行动，而不是一进来就把"立即安装"按钮推到最显眼的位置。这和很多 SaaS 工具网站"先把 CTA 做大"的思路形成对比，体现了对用户决策流程的尊重。

#### 核心设计亮点：Prompt 工作流透明化

详情页直接把这个技能包的 Prompt 工作流程——包括需求收集表、内容检索策略、行为逻辑——完整暴露给用户。这在普通 App Store 里是**极少见的**。

**设计意图**：这个市场的用户不是普通消费者，而是有能力阅读、评估甚至修改 Prompt 的 AI 工作者。透明化 Prompt 是建立专业用户信任的最直接方式，也帮助用户判断这个 skill 是否真的符合自己的场景，而不是依赖营销话术。

#### 体验设计精髓：需求收集表格

详情页那张 **7 行 3 列**的表格（信息 / 说明 / 默认值）是整个详情页最有价值的设计元素：

| 信息 | 说明 | 默认值 |
|------|------|--------|
| ... | ... | AI自动填 |

**设计价值**：
- 告诉用户这个技能能做什么
- 告诉用户"你什么都不填也没关系，AI 会用默认值帮你"
- **大幅降低上手门槛**，消除用户面对空白输入框时的不确定感
- 典型的**渐进式披露**设计思路，让新手和专家都能各取所需

#### 细节设计：多 AI 平台图标

安装引导卡片里排列了多个 AI 客户端的小图标，暗示这个技能包可以**跨平台使用**。

> 这个设计在视觉上只占很小的面积，但传递的信息量很大——它在告诉用户"这不是一个只为某个特定 AI 服务的技能，它是开放的"，这对提升平台吸引力非常重要。

### 一、页面头部信息区

### 二、安装区域

#### Install 面板

示例提示词区块
- 带格式的提示词内容展示
- "复制提示词"按钮（珊瑚色）

| 元素 | 说明 |
|------|------|
| **安装方式切换** | CLI / Prompt 两个 Tab |
| **命令行代码块** | `$ openclaw skills install @zhj7860/ppt` |
| **一键复制按钮** | 复制安装命令 |

### 三、内容标签页
根据 skill.md 做详细展示

如，四个标签：**SKILL.md** / **Skill Card** / **Files** / **Versions**

#### SKILL.md（默认激活）

完整技能文档，包含：

1. **设计哲学**：技能的设计理念
2. **生成流程**：Step 1-6 分步骤说明
3. **输出顺序**：输出内容的排列逻辑
4. **视觉规范速查表**：表格形式展示参数
5. **交互要求**：用户交互规范
6. **技术栈**：使用的技术栈说明
7. **严禁行为**：禁止的操作
8. **默认规则**：预设行为规则

#### 其他 Tab

| 标签 | 内容 |
|------|------|
| **Skill Card** | 技能卡片预览 |
| **Files** | 文件列表 |
| **Versions** | 版本历史 |
