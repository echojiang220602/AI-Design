## 一、整体页面布局

### 1.1 骨架结构

```
┌────────────────────────────┬─────────────────────────────┐
│  Card1：Agent 大标题卡片   │                            │
│  Card2：Agent实例卡片      │  人格设定 Tab 卡片         │
│  （左侧两卡片合并高度）     │  （高度与左侧相等）          │
├────────────────────────────┼─────────────────────────────┤
│  定时任务卡片               │  技能工具列表卡片           │
│  （与右侧等高）            │  （与左侧等高）            │
└────────────────────────────┴─────────────────────────────┘
```

**技术实现**：
- 容器：`max-w-[1400px] mx-auto py-6 px-4`
- 布局：CSS Grid，`grid-template-columns: 1fr 1fr`，`grid-template-rows: auto auto`
- 高度同步：JavaScript（双 `requestAnimationFrame` + `ResizeObserver`）
  - 人格设定卡片撑到左侧 Card1+2 合并高度，内容溢出时内部滚动
  - 底部两张卡片双向等高：谁高让对方匹配谁

### 1.2 高度同步详细逻辑

```tsx
// 1. 量 Card1 + Card2 的自然高度（去掉已有约束）
// 2. 设给人格设定 Card minHeight = 两者之和
// 3. 底部等高：blEl / brEl ResizeObserver 双向同步
```

**边界处理**：
- 窗口 resize：ResizeObserver 自动重算
- SSR / 无 JS 环境：降级为纯内容撑开布局
- 窄屏（<768px）：2 列 → 单列（定时任务 → 技能工具列表 → 人格设定 → Card2 → Card1）

## 二、定时任务卡片

### 2.1 布局结构

```
┌─────────────────────────────────┐
│  [Clock]  定时任务              │
├─────────────────────────────────┤
│  [FileText] 每日报告生成   [===]  │  ← Switch 开
│  0 9 * * * · 上次: 2小时前      │
│  [Code2]    代码质量检查   [===]  │  ← Switch 开
│  0 */4 * * * · 上次: 30分钟前    │
│  [Database]  数据备份       [  ]  │  ← Switch 关
│  0 2 * * * · 上次: 昨天          │
│  ...                              │
│  [ 查看更多 → ]                   │
└─────────────────────────────────┘
```

### 2.2 任务图标差异化

| 关键词 | 图标 | 颜色（启用） | 颜色（禁用） |
|---|---|---|---|
| 报告 / 总结 | `FileText` | `#0099DD` 蓝 | `#9CA3AF` 灰 |
| 代码 / 质量 | `Code2` | `#22C55E` 绿 | `#9CA3AF` 灰 |
| 备份 / 同步 / 清理 | `Database` | `#64748BS` 灰蓝 | `#9CA3AF` 灰 |
| 安全 / 监控 | `Shield` | `#EF4444` 红 | `#9CA3AF` 灰 |
| 健康检查 | `Activity` | `#F97316` 橙 | `#9CA3AF` 灰 |
| 其他 | `Clock` | `#0099DD` 蓝 | `#9CA3AF` 灰 |

**图标容器**：32px 圆形背景（`w-8 h-8 rounded-full`），`bg-primary-100`（启用）/ `bg-neutral-100`（禁用）

### 2.3 Switch 开关组件

```tsx
<button
  type="button"
  role="switch"
  aria-checked={checked}
  aria-label={taskName}
  onClick={() => onChange(!checked)}
  className="w-10 h-5 rounded-full transition-colors duration-200
    ${checked ? 'bg-[#0099DD]' : 'bg-neutral-300'}"
>
  <span className="w-4 h-4 bg-white rounded-full shadow
    transition-transform duration-200
    ${checked ? 'translate-x-[22px]' : 'translate-x-[2px]'}" />
</button>
```

**样式规格**：
| 状态 | 轨道色 | 滑块位置 |
|---|---|---|
| 开启（checked） | `#0099DD` 蓝 | `translate-x-[22px]` |
| 关闭 | `#D1D5DB` 灰 | `translate-x-[2px]` |

**无障碍**：`role="switch"` + `aria-checked` + `aria-label`（标注任务名称）

### 2.4 禁用状态

禁用任务：`opacity-50`，图标灰化，行可点击（Switch 仍可操作）

### 2.5 查看更多弹窗（TaskModal）

- 触发：点击「查看更多 →」按钮（`ViewMoreButton` 组件）
- 样式：全屏遮罩 `bg-black/50`，居中白卡 `rounded-xl w-[560px] max-h-[80vh]`
- 内容：全部任务列表 + Switch 均可操作
- 关闭：点击遮罩 / 点击 × 按钮

## 三、技能工具列表卡片

### 3.1 布局结构

```
┌─────────────────────────────────┐
│  [Wrench]  技能工具列表         │
├─────────────────────────────────┤
│  [Code2]  miaoda-coding    [安装]│  ← 技能（安装按钮）
│  [技能] 代码开发技能              │
│  [Database] miaoda-database [=] │  ← 工具（Switch）
│  [工具] 数据库操作工具            │
│  [Search]  web-search      [=] │
│  [工具] 网络搜索工具              │
│  ...                              │
│  [ 查看更多 → ]                   │
└─────────────────────────────────┘
```

### 3.2 技能与工具的区分

| 类型 | 区分方式 | 交互 |
|------|----------|------|
| 技能（skill） | 绿色标签 `bg-green-100 text-green-700` | 安装 / 已安装 / 卸载按钮 |
| 工具（tool） | 青色标签 `bg-cyan-100 text-cyan-700` | Switch 启用 / 禁用 |

**说明**：
- 技能：二态操作（安装/未安装），无法"禁用但不卸载"
- 工具：可以启用/禁用，但保留安装状态

### 3.3 图标差异化

| 关键词 | 图标 | 颜色（启用） | 颜色（禁用） |
|---|---|---|---|
| coding / 开发 | `Code2` | `#22C55E` 绿 | `#9CA3AF` 灰 |
| database / 数据库 | `Database` | `#64748BS` 灰蓝 | `#9CA3AF` 灰 |
| search / 搜索 | `Search` | `#3B82F6` 蓝 | `#9CA3AF` 灰 |
| analysis / 分析 | `TrendingUp` | `#8B5CF6` 紫 | `#9CA3AF` 灰 |
| file / 文件 | `Folder` | `#06B6D4` 青 | `#9CA3AF` 灰 |
| api / tester | `Zap` | `#F97316` 橙 | `#9CA3AF` 灰 |
| git | `GitBranch` | `#F97316` 橙 | `#9CA3AF` 灰 |
| 其他 | `Wrench` | `#0099DD` 蓝 | `#9CA3AF` 灰 |

### 3.4 技能列表项规格

```
┌─────────────────────────────────────────┐
│  [图标]  name（英文/不标准名称）         │
│          [技能] / [工具] 类型标签         │
│          description（可选，可能为空）     │
│                          [安装/已安装/卸载]│
└─────────────────────────────────────────┘
```

**说明**：
- `name`：原始名称，可能非中文或格式不标准，直接展示
- `displayName`（可选）：中文展示名，有则显示，无则用 `name`
- `description`（可选）：描述内容，无则显示"暂无描述"

### 3.5 安装按钮状态机

| 状态 | 文字 | 样式 | 可点击 |
|---|---|---|---|
| 未安装 | "安装" | `bg-[#0099DD] text-white` | 是 |
| 安装中 | "安装中..." | `bg-[#0099DD] text-white opacity-50` + Spinner | 否 |
| 已安装 | "已安装" | `bg-green-100 text-green-700` | 否 |
| 卸载 | 点击"已安装"展开下拉菜单 | - | 是 |

**卸载交互**：
- 点击"已安装"按钮 → 展开下拉菜单
- 选择「卸载」→ 确认弹窗 → 卸载成功 → 变为"安装"

### 3.6 工具 Switch 规格

同定时任务 Switch，禁用时 `opacity-50`。

### 3.7 展示顺序

| 优先级 | 排序规则 |
|--------|----------|
| 1 | 技能优先于工具 |
| 2 | 按安装时间倒序（最新安装的在前） |
| 3 | 工具按启用状态（启用的在前） |

### 3.8 查看更多弹窗（SkillModal）

结构同 TaskModal，全量技能/工具列表：
- 技能：安装按钮可操作
- 工具：Switch 可操作

## 四、交互规范

### 4.1 Switch 开关（定时任务 / 工具）

- 点击：立即更新前端状态（乐观更新）
- 行为：触发 `onChange` 回调，更新 `tasks[]` / `skills[]` 中的 `enabled` 字段
- 持久化：前端状态变更后，异步调用后端 API 同步

### 4.2 技能安装 / 卸载

| 操作 | 流程 | 反馈 |
|------|------|------|
| 安装 | 点击「安装」→ 变为「安装中...」→ 调用 API → 变为「已安装」 | Toast：安装成功 / 失败 |
| 卸载 | 点击「已安装」→ 展开下拉菜单 → 选择「卸载」→ 确认弹窗 → 调用 API → 变为「安装」 | Toast：卸载成功 / 失败 |

### 4.3 弹窗

- 打开：点击「查看更多 →」
- 关闭：点击遮罩层 / 点击 × 按钮
- 弹窗内操作：Switch 即时生效，同步影响主列表

## 五、数据结构与 API

### 5.1 数据结构

```tsx
// 定时任务
interface TaskItem {
  id: string;
  name: string;
  cron: string;          // Cron 表达式
  enabled: boolean;
  lastRunTime?: string;  // 上次运行时间
}

// 技能工具
interface SkillItem {
  id: string;
  name: string;           // 原始名称（可能非中文、不标准）
  displayName?: string;   // 可选中文展示名
  description?: string;     // 可选描述
  type: 'skill' | 'tool';
  status: 'installed' | 'not_installed';  // 技能：安装状态
  enabled?: boolean;         // 工具：启用状态
}
```

### 5.2 待定义 API

| 数据 | API 路径（待定） | 说明 |
|---|---|---|
| 定时任务列表 | `GET /api/agents/{id}/tasks` | 返回 `[{id, name, cron, enabled, lastRunTime}]` |
| 任务启用/禁用 | `PATCH /api/agents/{id}/tasks/{taskId}` | body: `{enabled: boolean}` |
| 技能列表 | `GET /api/agents/{id}/skills` | 返回 `[{id, name, displayName?, description?, type, status}]` |
| 技能安装 | `POST /api/agents/{id}/skills/{skillId}/install` | 安装技能 |
| 技能卸载 | `DELETE /api/agents/{id}/skills/{skillId}` | 卸载技能 |
| 工具启用/禁用 | `PATCH /api/agents/{id}/skills/{skillId}` | body: `{enabled: boolean}` |
