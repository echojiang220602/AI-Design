# 「我的Agent」页面整体设计方案

> 本文档是「我的Agent」页面从 0 到 1 的完整设计方案，包含产品结构、交互逻辑、视觉规范、技术实现和数据源说明。可作为 UI/UX 设计交付和前端开发规格的双向依据。

---

## 一、产品定位与用户场景

**用户**：拥有至少一个 AI 智能体（Agent）的终端用户

**核心场景**：
1. 查看 Agent 的基本信息和运行状态
2. 管理 Agent 的人格设定文件（SOUL.md / AGENT.md / IDENTITY.md / USER.md）
3. 查看定时任务列表，启用/禁用任务
4. 查看技能工具列表，启用/禁用工具
5. 发起与 Agent 的对话

**设计原则**：
- 信息密度适中：单页展示全部关键信息，无需跳转
- 操作即时反馈：Switch 点击立即响应，后端异步同步
- 左右对称美观：高度同步布局，避免空白或溢出

## 二、整体页面布局

### 2.1 骨架结构

```
┌────────────────────────────┬─────────────────────────────┐
│  Card1：Agent 大标题卡片   │                            │
│  Card2：Agent实例卡片      │  人格设定 Tab 卡片         │
│  （左侧两卡片合并高度）     │  （高度与左侧相等）          │
├────────────────────────────┼─────────────────────────────┤
│  定时任务卡片               │  技能工具列表卡片           │
│  （与右侧等高）            │  （与左侧等高）            │
└────────────────────────────┴─────────────────────────────┘
```

**技术实现**：
- 容器：`max-w-[1400px] mx-auto py-6 px-4`
- 布局：CSS Grid，`grid-template-columns: 1fr 1fr`，`grid-template-rows: auto auto`
- 高度同步：JavaScript（双 `requestAnimationFrame` + `ResizeObserver`）
  - 人格设定卡片撑到左侧 Card1+2 合并高度，内容溢出时内部滚动
  - 底部两张卡片双向等高：谁高让对方匹配谁

### 2.2 高度同步详细逻辑

```tsx
// 1. 量 Card1 + Card2 的自然高度（去掉已有约束）
// 2. 设给人格设定 Card minHeight = 两者之和
// 3. 底部等高：blEl / brEl ResizeObserver 双向同步
```

**边界处理**：
- 窗口 resize：ResizeObserver 自动重算
- SSR / 无 JS 环境：降级为纯内容撑开布局
- 窄屏（<768px）：2 列 → 单列（定时任务 → 技能工具列表 → 人格设定 → Card2 → Card1）

## 五、人格设定卡片（右侧）

### 5.1 Tab 导航

| Tab | 图标 | 对应文件 | 可编辑 | 警告 |
|---|---|---|---|---|
| 人格设定 | `FileText` | SOUL.md | ✅ | — |
| 用户档案 | `User` | USER.md | ✅ | — |
| 工作手册 | `Zap` | AGENT.md | ✅ | ⚠️ 非必要不建议修改 |
| 智能体档案 | `Bot` | IDENTITY.md | ✅ | ⚠️ 非必要不建议修改 |
| 全部文件 | `Folder` | — | — | 跳转 `/files` |

**Tab 样式**：
- 激活：蓝色下边框 `border-b-2 #0099DD`，浅蓝背景 `rgba(0,153,221,0.08)`
- 默认：灰色 `border-transparent`，透明背景
- 过渡：`transition-colors`
- 滚动：标签过多时横向滚动，隐藏默认滚动条

**交互**：
- 点击「全部文件」：路由跳转 `/files`（不切换 Tab 内容）
- 其他 Tab：切换 `activeTab` state，渲染对应内容

### 5.2 内容区 — FileTabContent 组件

**Props**：
```tsx
interface FileTabContentProps {
  filename: string;      // 文件名（SOUL.md / AGENT.md / ...）
  title: string;         // 文件说明
  description: string;   // 副标题
  showEditButton?: boolean;
  previewMode?: boolean;  // 内容超长时截断预览
  content: string;        // 文件内容（Markdown）
  onEdit?: (content: string) => void;
}
```

**状态**：
- `isEditing: boolean` — 是否编辑模式
- `editContent: string` — 编辑中的内容

**交互流程**：
1. 预览模式（默认）：显示内容前 100 字符 + `...`，点击「编辑」进入编辑模式
2. 编辑模式：textarea 替换内容区，点击「保存」调用 `onEdit`，点击「取消」还原
3. 保存后：重新回到预览模式

**Markdown 渲染**：使用 `<pre>` 纯展示（当前实现），未来可升级为 `react-markdown`

### 6.2 任务图标差异化

| 关键词 | 图标 | 颜色（启用） | 颜色（禁用） |
|---|---|---|---|
| 报告 / 总结 | `FileText` | `#0099DD` 蓝 | `#9CA3AF` 灰 |
| 代码 / 质量 | `Code2` | `#22C55E` 绿 | `#9CA3AF` 灰 |

**图标容器**：32px 圆形背景（`w-8 h-8 rounded-full`），`bg-primary-100`（启用）/ `bg-neutral-100`（禁用）

### 6.3 Switch 开关组件

```tsx
<button
  type="button"
  role="switch"
  aria-checked={checked}
  aria-label={taskName}
  onClick={() => onChange(!checked)}
  className="w-10 h-5 rounded-full transition-colors duration-200
    ${checked ? 'bg-[#0099DD]' : 'bg-neutral-300'}"
>
  <span className="w-4 h-4 bg-white rounded-full shadow
    transition-transform duration-200
    ${checked ? 'translate-x-[22px]' : 'translate-x-[2px]'}" />
</button>
```

**样式规格**：
| 状态 | 轨道色 | 滑块位置 |
|---|---|---|
| 开启（checked） | `#0099DD` 蓝 | `translate-x-[22px]` |
| 关闭 | `#D1D5DB` 灰 | `translate-x-[2px]` |

**无障碍**：`role="switch"` + `aria-checked` + `aria-label`（标注任务名称）

### 6.4 禁用状态

禁用任务：`opacity-50`，图标灰化，行可点击（Switch 仍可操作）

### 6.5 查看更多弹窗（TaskModal）

- 触发：点击「查看更多 →」按钮（`ViewMoreButton` 组件）
- 样式：全屏遮罩 `bg-black/50`，居中白卡 `rounded-xl w-[560px] max-h-[80vh]`
- 内容：全部 8 个任务列表 + Switch 均可操作
- 关闭：点击遮罩 / 点击 × 按钮

### 7.2 技能与工具的区分

| 类型 | 区分方式 | 交互 |
|------|----------|------|
| 技能（skill） | 绿色标签 `bg-green-100 text-green-700` | 安装 / 已安装 / 卸载按钮 |

**说明**：
- 技能：二态操作（安装/未安装），无法"禁用但不卸载"

### 7.4 技能列表项规格

```
┌─────────────────────────────────────────┐
│  [图标]  name（英文/不标准名称）         │
│          [技能] / [工具] 类型标签         │
│          description（可选，可能为空）     │
```

**说明**：
- `name`：原始名称，可能非中文或格式不标准，直接展示
- `displayName`（可选）：中文展示名，有则显示，无则用 `name`
- `description`（可选）：描述内容，无则显示"暂无描述"

### 7.5 安装按钮状态机

| 状态 | 文字 | 样式 | 可点击 |
|---|---|---|---|
| 未安装 | "安装" | `bg-[#0099DD] text-white` | 是 |
| 安装中 | "安装中..." | `bg-[#0099DD] text-white opacity-50` + Spinner | 否 |
| 已安装 | "已安装" | `bg-green-100 text-green-700` | 否 |
| 卸载 | 点击"已安装"展开下拉菜单 | - | 是 |

- 点击"已安装"按钮 → 展开下拉菜单
- 选择「卸载」→ 确认弹窗 → 卸载成功 → 变为"安装"

### 7.6 工具 Switch 规格

同定时任务 Switch，禁用时 `opacity-50`。

### 7.7 展示顺序

| 优先级 | 排序规则 |
|--------|----------|
| 1 | 技能优先于工具 |
| 2 | 按安装时间倒序（最新安装的在前） |
| 3 | 工具按启用状态（启用的在前） |

### 7.8 查看更多弹窗（SkillModal）

结构同 TaskModal，全量技能/工具列表：
- 技能：安装按钮可操作
- 工具：Switch 可操作

## 九、交互规范

### 9.1 Switch 开关（定时任务 / 工具）

- 点击：立即更新前端状态（乐观更新）
- 行为：触发 `onChange` 回调，更新 `tasks[]` / `skills[]` 中的 `enabled` 字段
- 持久化：前端状态变更后，异步调用后端 API 同步（见数据源章节）

### 9.2 技能安装 / 卸载

| 操作 | 流程 | 反馈 |
|------|------|------|
| 安装 | 点击「安装」→ 变为「安装中...」→ 调用 API → 变为「已安装」 | Toast：安装成功 / 失败 |
| 卸载 | 点击「已安装」→ 展开下拉菜单 → 选择「卸载」→ 确认弹窗 → 调用 API → 变为「安装」 | Toast：卸载成功 / 失败 |

### 9.2 弹窗

- 打开：点击「查看更多 →」
- 关闭：点击遮罩层 / 点击 × 按钮
- 弹窗内操作：Switch 即时生效，同步影响主列表（如弹窗内禁用，主列表同步灰化）

### 9.3 Tab 切换

- 点击：立即切换，无动画
- 键盘：方向键切换（需 `tabindex` 支持）

### 9.4 热力图

- hover：月份标签 + 星期标签渐显（`opacity-0 → 1`，200ms）
- 鼠标移出：标签渐隐

## 十、技术实现摘要

### 10.1 技术栈

- **框架**：React 18 + TypeScript（严格模式）
- **路由**：React Router v6
- **样式**：Tailwind CSS（严禁 CSS Modules / styled-components）
- **图标**：Lucide React（多尺寸：28px / 24px / 20px / 16px / 12px）
- **构建**：Vite 5 + pnpm

### 10.2 目录结构

```
src/
└── pages/shrimp/pages/AgentPage.tsx    ← 主页面（所有组件内联）
    └── components/
        └── agent/
            └── ViewMoreButton.tsx       ← 查看更多按钮
        └── common/
            ├── Card.tsx                 ← 卡片组件（forwardRef）
            ├── Button.tsx

```
