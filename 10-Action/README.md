# 10-Action/ 执行层

> 回答一个核心问题：**现在要做的事是什么，下一步具体怎么做？**

这里存放所有正在推进的任务和项目。每一件「要做的事」对应一则任务笔记，所有相关信息都在那里管理。

---

## 结构

| 子文件夹 | 用途 |
|---------|------|
| `11-Focus-聚焦承诺/` | 当前最重要的 1-3 个核心项目，每次只聚焦这里 |
| `12-Rearch-调研分析/` | 调研分析、对比研究、参考笔记 |
| `13-Maybe-将来也许/` | 感兴趣但暂时不做、「有空再说」的想法 |
| `14-design-产品设计/` | 产品设计方案文档（含子文件夹：1400-整体规划/1401-创建Agent/1403-我的Agent/1404-Skill广场/1405-专项Agent） |
| `15-project-项目跟进/` | 项目跟进、会议纪要、客户需求分析 |
| `16-present-发布会/` | 发布会相关文档（演讲稿、星际图、字幕等） |

---

## 任务笔记结构

一则完整的任务笔记包含：

```markdown
# 任务名称

## 任务背景
这件事从哪来的，谁提出的，为什么要做？

## 任务成果
完成了能拿到什么，怎么验证完成了？

## 相关资料
- 经过处理的资料（用自己的话概括，写清楚为什么重要）
- 不是原文复制，是你消化后的理解

## 行动清单
- [ ] 下一步要做的具体事（优先级高的在前）

## 进展记录
日期 + 当天推进了什么，卡在哪里，下次从哪里继续
```

**关键原则**：每条资料都要经过处理——用自己的话概括 + 写出下一步行动。不要把任务笔记变成收藏夹。

---

## 任务命名规范

格式：`专有名词 + 任务属性`

- 让你一眼认出这是哪件事、这则笔记是什么
- 示例：`课程制作-第七节逐字稿`、`王总-Q3策略方案`

---

## 与其他层的连接

- 任务笔记的顶部应有链接指向 **Time 层**的目标（`30-Time/32-12Week/`）
- 任务笔记引用的卡片用 `[[卡片名]]` 双链连接（来自 `20-Card/`）

---

## 子文件夹命名规范

**统一格式**：`<前缀>-<4位编号>-<描述>.md`，前缀取**最深文件夹**的数字前缀，编号固定 4 位（0001-9999）。

| 文件夹 | 命名格式 | 示例 |
|--------|---------|------|
| `11-Focus-聚焦承诺/` | 项目文件夹内自由命名 | `AI数字人中台/` |
| `12-Rearch-调研分析/` | `12-NNNN-描述.md` | `12-0001-AI项目工作拆分.md` |
| `13-Maybe-将来也许/` | `M日期-描述.md`（暂未完全规范化） | `M260101-潜在合作方向.md` |
| `14-design-产品设计/` | 容器，子文件夹分主题 | `1400-整体规划/`、`1403-我的Agent/` |
| └ 子文件夹内 | `14NN-NNNN-描述.md` | `1400-0001-AI数字人-技术方案.md` |
| `15-project-项目跟进/` | 容器，子文件夹分主题 | `1500-项目整体情况/`、`1501-高价值数据/`、`1506-温台项目/` |
| └ 子文件夹内 | `15NN-NNNN-描述.md` | `1506-0001-0625温台项目会议.md` |
| `16-present-发布会/` | `16-NNNN-描述.md` | `16-0001-AI数字员工发布会.md` |

**重命名时必须同步更新所有引用**（wikilink / 纯文本 / YAML / 代码块），详见 [[日常-md重命名]] Skill。
